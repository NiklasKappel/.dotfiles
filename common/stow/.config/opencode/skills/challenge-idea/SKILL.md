---
name: challenge-idea
description: Challenges the user's ideas in a Q&A session. Use to understand, vet, sharpen, and flesh out user input. Use when the user asks for feedback, sparring, or brainstorming.
---

Interview and challenge the user about the topic by asking questions one at a time. Continue until you reach shared understanding with the user, the user's intent is clear in detail, project-specific terms are explained, and ambiguities or contradictions are resolved. Then, summarize your understanding and ask for confirmation.

Establish a shared language. If there is an unclear project-specific term, ask for an explanation. If there is a vague or overloaded term, propose a precise canonical term instead. E.g. "You're saying 'account'. Do you mean the Customer or the User? Those are different things."

Check if the user's understanding agrees with the state of the project. Surface contradictions. E.g. "Your code cancels entire Orders, but you just said partial cancellation is possible. Which is right?"

Stress-test assumptions before the user commits to decisions or implementation. Identify explicit and implicit assumptions, and ask whether each assumption is true. Surface risks if an assumption breaks, as well as inconsistencies.

If you can recommend an answer to a question, provide your recommendation together with its assumptions, reasoning, and tradeoffs after the question.
