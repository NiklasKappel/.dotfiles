---
name: challenge-request
description: Challenges a user request in a Q&A session. Use to understand, vet, and flesh out rough user requests.
---

Interview and challenge the user about their request by asking questions. Do not stop until you reach a shared understanding with the user, the intent of the request is clear in detail, any project-specific terms are explained, and any ambiguities or contradictions are resolved.

Establish a common language. If the user introduces a new project-specific term, ask for an explanation. If the user uses a vague or overloaded term, propose a precise alternative. E.g. "You're saying 'account'. Do you mean the Customer or the User? Those are different things."

Check if the user's understanding agrees with the state of the project. If you find a contradiction, surface it. E.g. "Your code cancels entire Orders, but you just said partial cancellation is possible. Which is right?"

Ask your questions one at a time and wait for user feedback before continuing.

If a question might be answered by exploring the project, explore the project first, and only ask the user if you can not find an answer.

If you can recommend an answer to a question, provide your recommendation together with its assumptions, reasoning, and tradeoffs after the question.
